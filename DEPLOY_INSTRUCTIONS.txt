
PASSO A PASSO PARA CORREÇÃO NO VPS:

1. Atualize o Nginx:
   sudo nano /etc/nginx/sites-available/gestaokavins.com.br
   (Apague o conteúdo antigo e cole o novo NGINX_CONFIG.conf)

2. Reinicie o Nginx:
   sudo nginx -t && sudo systemctl restart nginx

3. No projeto CORTE:
   # Certifique-se que o build está atualizado
   cd /var/www/gestaokavins/corte
   npm run build
   
   # Reinicie o processo no PM2
   pm2 restart kavins-corte
   # Caso ainda não exista: pm2 start server.js --name "kavins-corte"

4. No projeto PRINCIPAL:
   # Garanta que ele está rodando na porta correta (ex: 3001) se usar API
   pm2 restart kavins-principal

DICA: Se ao acessar /corte ele ainda carregar o antigo, limpe o cache do navegador (Ctrl + F5).
O uso de 'proxy_pass' para a porta 3002 isola totalmente os dois ambientes.
